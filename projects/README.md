# /projects

**Purpose:** Ephemeral workspaces for ongoing sprints, features, or audits.

**Rules:**
1. Active work MUST happen inside `/projects/active/<project-name>/`.
2. Do not write scratch code or WIP files to the root workspace.
3. Sub-agents and temporary agents are granted write access here.
4. When a sprint concludes, artifacts should be merged into `/domains/` or `/apps/`, and the project folder moved to `/projects/archive/`.
