### 1.3 `Asset light Strategy .md` — Asset-Light Model

#### Strategic Shift
**From:** Owning flow, holding stock, repacking, absorbing damage, fixed warehouse cost
**To:** Designing flow, enforcing rules, orchestrating suppliers → hubs → customers, taking margin on coordination

#### Three Core Flows
1. **SuperLeader Direct Cluster:** Farm → Supplier DC → Super Leader → B2C Pickup (no central warehouse)
2. **Direct DC → Restaurants:** Supplier/Aggregator DC → Restaurant (no ChipChip touch)
3. **FMCG Direct Drop-Ship:** Supplier-held, delivered only against confirmed demand, zero pre-buy inventory

#### Key Design Principle
- Push physical risk outward (supplier absorbs handling, sorting, storage)
- Keep digital + rule control inward (pricing, ordering, settlement, SLA enforcement)

#### Non-Negotiable Rules (Without Warehouse)
1. **No Same-Day Flexibility** — everything has a cutoff, no last-minute adds
2. **Volume Commitment Windows** — SLs and restaurants must commit/lock orders
3. **Supplier Liability Shift** — damage/late/wrong = supplier absorbs first layer
4. **CP1 Auto-Block** — no order below margin floor

#### Company Structure Vision
- **Layer 1:** Demand Aggregation
- **Layer 2:** Credit Intelligence
- **Layer 3:** Enforcement Engine

#### Future Revenue Streams
- Margin on goods (transaction revenue)
- Financing services (financial revenue) — micro-credit, farmer financing, restaurant BNPL
- Supplier tools/ads (SaaS revenue)

#### Blockchain Use Cases
1. **Trade Credit Ledger** — immutable B2B transaction records, portable credit profiles
2. **Supplier SLA Proof** — digital delivery confirmation, clear liability assignment
3. **Trust + dispute + proof + default risk** — public trust layer

### 1.4 `Packaging .md` — Packaging System Design

#### Bag System (Deposit/Refund)
- **New costs:** Big bag 30 ETB, Mid bag 15 ETB (3.5× increase from old system)
- **Recommended model:** Deposit-based return OR Packaging Wallet Credit
- Deposit refunded only after scan confirmation (no manual cash refund)
- 7-day return window
- Wallet credit model encourages repeat orders

#### Crate System (Asset Control)
- Crates tracked as inventory with driver liability ledger
- Trip-based reconciliation: warehouse issues → driver delivers → end-of-day return
- Mismatch = "crate debt" → no next-day trips until resolved
- Asset IDs (QR/barcode) required

#### Hybrid Model
- **B2B:** Full crate exchange, mandatory, next delivery blocked if no return
- **Super Leaders:** Crate pool model, SL responsible, deposit at SL level
- **Direct B2C:** Bag deposit, encourage migration to SL

#### Critical Metrics
1. Crate loss rate %
2. Bag reuse rate %
3. Refund fraud rate
4. Driver crate balance
5. Average crate cycle time

#### Strategic Insight
Packaging system supports profitability: reduces tomato damage %, reduces warehouse OT, increases route density, pushes migration to Super Leaders.

### 2.3 `paperclip-deep-dive.md` + `research-paperclip.md` + `paperclip-integration-guide.md` — Paperclip Analysis

#### What Paperclip Is
AI company orchestrator — manages AI agents as employees with org charts, budgets, governance, goals, heartbeats, ticketing. Node.js + React + embedded PostgreSQL. MIT license.

#### Key Architecture
- **Org chart:** Self-referential `reportsTo` FK creates tree-structured hierarchy
- **Budget enforcement:** 3-tier (company → agent → project), warn at 80%, hard stop at 100%
- **Atomic task checkout:** `checkoutRunId` + `executionLockedAt` prevents double-work
- **Goal alignment:** Company Mission → Project Goal → Task (every task traces to "why")
- **Heartbeats:** Scheduled agent wake-up with context delivery (fat vs thin payload)
- **Adapters:** OpenClaw gateway (WebSocket), Claude, Codex, Cursor, Bash, HTTP

```
Bruk (Board/Owner)
```

#### Supplier Onboarding SOP (CHIP-OPS-002)
- **5-stage pipeline:** Sourcing (Days 1-3) → Documentation (Days 4-10) → Quality Evaluation (Days 8-14) → Agreement (Days 12-18) → Training & Go-Live (Days 16-21)

- QA scoring: 1-5 scale, minimum 3.5 average, no single criterion below 3.0

- **SLAs:** Order confirmation <30s, supplier notification <15min, supplier readiness <2h
- **Flow:** Receipt → Validation → Consolidation → Supplier Fulfillment → Delivery → Post-Delivery

#### Quality Assurance SOP (CHIP-OPS-004)
- **4 quality checkpoints:** Supplier pre-shipment → Hub intake (10% sampling) → Pre-delivery spot check → Customer receipt
- **Supplier quality scoring:** 0-100 scale, 5 tiers (Platinum 90-100 → Suspended <50)
- **Non-conformance:** Minor (document only), Moderate (NCR + corrective action), Major (immediate quarantine + suspension)

#### Emergency Response SOP (CHIP-OPS-006)
- **4 severity levels:** SEV-1 (Critical, 15min response) through SEV-4 (Minor, 4hr response)
- **6-person ERT:** Incident Commander, Technical Lead, Communications Lead, Customer Success Lead, Logistics Lead, Legal Advisor
