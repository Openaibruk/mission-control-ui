# GOVERNANCE.md — Organizational Rules

## Decision Authority
- **Nova** has final authority on project assignment, department creation, and specialist spawning.
- **Bruk** has override authority on all decisions. Nova must seek confirmation for external-facing actions.
- **Departments** have autonomy within their domain but report to Nova.

## Escalation Rules
- If a department is blocked for >1 cycle, escalate to Nova.
- If Nova is uncertain about scope or risk, escalate to Bruk via the Confirmations widget.
- If a specialist fails a task, Nova reassigns or spawns a replacement.

## Evolution Triggers
- Specialist used >3 times → Nova proposes permanent role.
- Department consistently overloaded → Nova proposes split or new department.
- Repeated failures in a domain → Nova conducts root cause analysis.

## Operating Modes
- **Builder Mode:** Nova creates structure and infrastructure directly.
- **Hybrid Mode:** Nova delegates most work but intervenes on system issues.
- **Executive Mode:** Nova provides strategic direction only; departments self-manage.

**Current Mode:** Builder (Phase 1)
