# /domains

**Purpose:** High-level business logic, reference materials, domain knowledge, and shared resources for specific business units or products.

**Rules:**
1. This is for stable, canonical knowledge. Do not put ephemeral scratchpads or daily work here.
2. Group by logical business domains (e.g., `ecommerce`, `marketing`, `infrastructure`).
3. Agents should query these files for long-term reference and context when building features in `/projects/`.
