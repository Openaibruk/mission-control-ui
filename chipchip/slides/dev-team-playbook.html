Sprint Planning Workflow

📋 Before Sprint

  • Nova: Review backlog, prioritize
  • Nova: Create sprint goal
  • Nova: Generate sprint tasks
  • Kiro: Flag arch dependencies

⚙️ During Sprint

  • Henok: Pick highest-priority task
  • Henok: Implement → in_progress → done
  • Cinder: Review completed work
  • Nova: Monitor blockers

🔄 After Sprint

How to Use This Playbook

🆕 New Feature Request

  1. Determine track size:
    • Quick Flow (1-5 pts) → Template 2
    • Full Feature (10+ pts) → Template 1
  2. Create project in Mission Control
  3. Create tasks following template steps
  4. Assign to right agent per mapping

🐛 Bug Report

  1. Use Template 3 (Bug Fix)
  2. Log bug as task with severity:
    • critical / high / medium / low
  3. Assign to Cinder (QA) first

🏗️ Tech Decision

  1. Use Template 4 (ADR)
  2. Create task with [ADR] prefix
  3. Assign to Kiro (Architect)
  4. Security review by Onyx if applicable
