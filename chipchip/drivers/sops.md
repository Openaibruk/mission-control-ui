## SOP-001: New Driver Registration

**Objective:** Process new driver applications efficiently and compliantly.

### Procedure

1. **Application Receipt**
   - Driver submits application via app or in-person
   - System generates unique Driver ID (CC-XXXX)
   - Confirmation SMS/email sent to driver

2. **Document Verification** (within 24h)
   - Verify driver's license authenticity
   - Check vehicle registration validity
   - Confirm insurance coverage
   - Run background check via approved provider
   - Flag any discrepancies → escalate to Operations Manager

3. **Eligibility Decision**
   - **Approved:** Proceed to account setup notification
   - **Conditional:** Request missing/incorrect documents
   - **Rejected:** Send rejection notification with reason and appeal process

4. **Account Provisioning**
   - Create driver profile in system
   - Assign to delivery zone based on address
   - Set driver tier to Bronze
   - Enable training module access

### Escalation Path
- Document verification issues → Operations Lead (within 4h)
- Background check flags → Compliance Officer (immediate)
- System errors → Tech Support (within 1h)

## SOP-002: Training & Assessment

**Objective:** Ensure all drivers are competent before activation.

### Procedure

1. **Training Access**
   - Send app download link + training login credentials
   - Schedule orientation day (within 7 days of approval)
   - Assign training materials (this manual)

2. **Training Completion Tracking**
   - Monitor module completion in admin dashboard
   - Send reminder if not started within 48h
   - Auto-unlock quiz after all modules completed

3. **Assessment**
   - Administer training quiz (minimum 80% to pass)
   - If failed: 24h cooldown, then retake allowed
   - Maximum 3 attempts → mandatory in-person review
   - Record score and completion date

4. **Practical Assessment**
   - Schedule supervised delivery (optional but recommended)
   - Evaluate: navigation, customer interaction, package handling
   - Document feedback in driver profile

### Escalation Path
- Training not started in 72h → Driver Support outreach
- Failed 3 quiz attempts → Operations Manager review
- Poor practical assessment → Additional training required

## SOP-003: First Week Driver Support

**Objective:** Maximize new driver success and retention during critical first week.

### Procedure

1. **Day 3 Check-in**
   - Automated message: "How's it going? Any questions?"
   - Review first delivery metrics (time, rating, issues)
   - If no deliveries completed → phone call from support

2. **Day 5 Performance Review**
   - Pull metrics: deliveries completed, avg rating, response time
   - If rating < 3.5: schedule coaching call
   - If 0 deliveries: investigate and resolve barriers

3. **Day 7 Week 1 Review**
   - Comprehensive performance summary
   - Identify strengths and improvement areas
   - Set goals for week 2
   - If below minimums: extend probation or discuss exit

### Minimum Week 1 Standards
- 5+ deliveries completed
- Average rating ≥ 4.0
- Delivery success rate ≥ 95%
- Response time < 2 minutes

### Escalation Path
- Below minimums → Extended probation (additional 7 days)
- Safety violations → Immediate suspension
- Driver requests to leave → Exit interview

## SOP-004: Driver Activation

**Objective:** Transition trained drivers to full operational status.

### Procedure

1. **Eligibility Check**
   - All onboarding phases completed
   - Training quiz passed
   - Week 1 minimums met
   - No outstanding compliance issues

2. **System Updates**
   - Remove "new driver" flag
   - Expand delivery zone access
   - Enable full feature set (peak hours, long-distance)
   - Add to active driver pool for assignments

3. **Community Onboarding**
   - Add to driver WhatsApp/Telegram group
   - Invite to next community meetup
   - Assign driver mentor (Gold/Platinum tier)

4. **Notification**
   - Send activation confirmation to driver
   - Notify team lead of new active driver
   - Log activation date for performance tracking

## SOP-005: Incident Handling During Onboarding

**Objective:** Handle issues that arise during the onboarding process.

### Procedure

1. **Incident Types**
   - Document fraud detected
   - Safety violation during training
   - Vehicle inspection failure
   - Customer complaint during supervised delivery
   - Driver injury during training

2. **Immediate Response**
   - Suspend onboarding process
   - Document incident details (time, location, people, description)
   - Notify Operations Manager within 1 hour
   - Ensure driver safety (medical attention if needed)

3. **Investigation**
   - Gather facts (witnesses, evidence, app data)
   - Determine severity level (Low/Medium/High/Critical)
   - Document findings in incident report

4. **Resolution**
   - **Low:** Warning + additional training
   - **Medium:** Extended probation + monitoring
   - **High:** Suspension pending review
   - **Critical:** Immediate termination + legal if needed

5. **Follow-up**
   - Update procedures if systemic issue found
   - Communicate outcome to relevant parties
   - File incident report in records

## SOP-006: Driver Exit Process

**Objective:** Handle driver departures professionally and securely.

### Procedure

1. **Voluntary Exit**
   - Receive notice (in-app, phone, or in-person)
   - Conduct exit interview (reasons, feedback, suggestions)
   - Process final payment (within 7 days)
   - Deactivate account and reclaim materials

2. **Involuntary Exit (Performance)**
   - Document performance issues with evidence
   - Issue 2 written warnings (7-day intervals)
   - Final review meeting with Operations Manager
   - Deactivate account, process final payment

3. **Involuntary Exit (Violation)**
   - Immediate suspension for safety/legal violations
   - Investigation within 48 hours
   - Decision: reinstatement with conditions or termination
   - Process final payment (minus any penalties)

4. **Post-Exit**
   - Remove from all driver groups
   - Reclaim physical materials (ID badge, stickers)
   - Archive driver records (retain for 2 years)
   - Update driver database
